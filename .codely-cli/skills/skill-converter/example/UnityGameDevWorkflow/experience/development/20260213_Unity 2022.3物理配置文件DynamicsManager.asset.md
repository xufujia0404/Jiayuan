## 问题描述

### 问题表现
Unity编辑器脚本执行时抛出异常：`无法加载PhysicsManager.asset`，提示文件不存在。

### 触发条件
- 使用 `AssetDatabase.LoadAllAssetsAtPath("ProjectSettings/PhysicsManager.asset")` 加载物理配置
- Unity版本为2022.3.x
- 尝试通过SerializedObject修改Physics全局参数

## 解决方案

### 关键步骤
1. 检查ProjectSettings目录下的实际文件名称
2. 将路径从 `ProjectSettings/PhysicsManager.asset` 修改为 `ProjectSettings/DynamicsManager.asset`
3. 使用正确的路径加载配置文件

### 关键代码/命令

```csharp
// ❌ 错误写法 - Unity 2022.3中PhysicsManager.asset不存在
var physicsAssets = AssetDatabase.LoadAllAssetsAtPath("ProjectSettings/PhysicsManager.asset");

// ✅ 正确写法 - 使用DynamicsManager.asset
var physicsAssets = AssetDatabase.LoadAllAssetsAtPath("ProjectSettings/DynamicsManager.asset");
if (physicsAssets != null && physicsAssets.Length > 0)
{
    SerializedObject physicsSettings = new SerializedObject(physicsAssets[0]);
    // 配置物理参数
    physicsSettings.ApplyModifiedProperties();
}
```

### 最终方案
在Unity 2022.3及更高版本中，物理配置文件的名称从 `PhysicsManager.asset` 变更为 `DynamicsManager.asset`。使用SerializedObject配置Physics全局参数时，必须使用正确的文件路径 `ProjectSettings/DynamicsManager.asset`。

**版本差异**：
- Unity 2021.x及更早版本：使用 `PhysicsManager.asset`
- Unity 2022.x及更高版本：使用 `DynamicsManager.asset`

## 问题描述

### 问题表现
Unity 编译失败，控制台输出异常：
`Library\ScriptAssemblies\Assembly-CSharp.dll: Copying the file failed: The process cannot access the file because it is being used by another process.`

### 触发条件
- 频繁执行 `execute_csharp_script` 脚本
- 脚本执行后立即触发编译
- Unity Editor 或相关进程（如 Roslyn/Bee）锁定 DLL 文件

## 解决方案

### 关键步骤
1. 等待 Unity Editor 空闲 (`unity_editor.wait_for_idle`)。
2. 重新触发编译 (`unity_editor.start_compilation_pipeline`)。
3. 如果问题持续，尝试重启 Unity Editor 或手动删除 `Library/ScriptAssemblies` 目录（但在自动化环境中只能重试）。

### 关键代码/命令
```python
# 遇到文件锁定错误时，先等待空闲
unity_editor.wait_for_idle()

# 然后重新请求编译
unity_editor.start_compilation_pipeline()
```

### 最终方案
这是一个瞬态错误，通常是由于文件系统锁定或进程竞争导致的。简单的重试策略（等待空闲后重试）通常能解决问题。避免在脚本执行未完全结束时立即触发编译。

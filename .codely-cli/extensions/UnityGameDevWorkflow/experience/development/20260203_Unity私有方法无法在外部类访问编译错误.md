## 问题描述

### 问题表现
Unity编译器报错：`'ClassName.MethodName' is inaccessible due to its protection level`

### 触发条件
在C#脚本中尝试从外部类访问另一个类的私有（private）方法时触发。

## 解决方案

### 关键步骤
1. 检查编译错误信息，定位到访问私有方法的代码行
2. 将需要在外部访问的方法的访问修饰符从 private 改为 public
3. 重新编译验证

### 关键代码/命令
```csharp
// 错误写法 - 私有方法
private Vector2Int GetGridPosition(Vector3 worldPosition)
{
    // ...
}

// 正确写法 - 公共方法
public Vector2Int GetGridPosition(Vector3 worldPosition)
{
    // ...
}
```

### 最终方案
当一个方法需要被其他类访问时，必须将其访问修饰符设置为 public。如果方法只在类内部使用，可以保持为 private。在 Unity 游戏开发中，控制器类通常需要暴露一些公共方法供其他管理器类调用，例如获取位置、检查碰撞等功能方法。根据实际需求选择合适的访问级别：public（任何地方可访问）、internal（同一程序集可访问）、protected（派生类可访问）、private（仅本类可访问）。

## 问题描述

### 问题表现
游戏启动后点击Play，场景中看不到关键游戏对象（如玩家、道具等），控制台显示游戏已初始化但对象未显示。

### 触发条件
- 在编辑器模式下创建场景对象时，为防止脚本过早执行而禁用对象（SetActive(false)）
- GameManager或其他管理脚本在游戏开始时未正确启用这些对象
- 对象在场景层级中存在但activeSelf为false

## 解决方案

### 关键步骤
1. 在GameManager的StartGame()方法中添加启用场景对象的逻辑
2. 使用GameObject.Find()查找场景中的关键对象
3. 检查对象是否被禁用，如果是则调用SetActive(true)
4. 确保在游戏循环开始前完成对象启用

### 关键代码/命令
```csharp
private GameObject sceneObject;

private void EnableSceneObjects()
{
    // 查找场景中的对象
    if (sceneObject == null)
    {
        GameObject obj = GameObject.Find("ObjectName");
        if (obj != null)
        {
            sceneObject = obj;
        }
    }
    
    // 启用对象
    if (sceneObject != null && !sceneObject.activeSelf)
    {
        sceneObject.SetActive(true);
    }
}
```

### 最终方案
在GameManager的StartGame()方法中调用EnableSceneObjects()，确保所有被禁用的关键游戏对象在游戏开始前被启用。对于预制体实例化的对象，在Instantiate后立即设置activeSelf为true。建议在场景搭建阶段避免禁用关键对象，或使用DontDestroyOnLoad标记持久化对象。

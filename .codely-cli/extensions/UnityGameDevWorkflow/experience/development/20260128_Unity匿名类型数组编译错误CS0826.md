## 问题描述

### 问题表现
Unity编辑器报告编译错误：`error CS0826: No best type found for implicitly-typed array`

### 触发条件
- 使用匿名类型创建数组并初始化
- 数组元素包含不同属性组合（如isTransparent、isParticles等可选属性）
- C#编译器无法推断数组元素类型

## 解决方案

### 关键步骤
1. 定义具名内部类（private class）替代匿名类型
2. 将匿名类型数组改为具名类数组
3. 确保所有数组元素使用相同的具名类

### 关键代码/命令
```csharp
// ❌ 错误方式（匿名类型数组）
var materialsInfo = new[]
{
    new { name = "Ball", color = new Color(1, 1, 1, 1), metallic = 1.0f, path = "..." },
    new { name = "Floor", color = new Color(0.5f, 0.5f, 0.5f, 1), alpha = 0.2f, path = "...", isTransparent = true }
};

// ✅ 正确方式（使用具名类）
private class MaterialInfo
{
    public string name;
    public Color color;
    public float metallic;
    public float smoothness;
    public float emission;
    public float alpha;
    public string path;
    public bool isTransparent;
    public bool isParticles;
}

var materialsInfo = new MaterialInfo[]
{
    new MaterialInfo { name = "Ball", color = new Color(1, 1, 1, 1), metallic = 1.0f, path = "..." },
    new MaterialInfo { name = "Floor", color = new Color(0.5f, 0.5f, 0.5f, 1), alpha = 0.2f, path = "...", isTransparent = true }
};
```

### 最终方案
使用具名内部类替代匿名类型数组，确保编译器能够正确推断数组元素类型。当数组元素包含不同属性组合时，具名类是唯一可靠的选择。

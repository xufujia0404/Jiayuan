# Unity 游戏调试执行器（Debugger）

此技能负责从经验知识库中检索相关内容，为用户提供问题定位与修复建议。

## 核心工作流

### Step 1: 理解用户查询

- 用户遇到的问题类型
- 关键词和术语
- 上下文信息
- 期望的解决方案类型

### Step 2: 搜索经验文档（必须先执行）

当用户提出游戏生成/开发中的 debug 需求时，先在以下目录检索：

- `experience/development/`

#### 搜索范围

- 文档标题
- 问题描述
- 触发条件
- 解决方案

#### ⚠️ 数量限制（重要）

- **最多读取不超过5个最相关的经验文档**
- 优先选择与当前任务最直接相关的文档
- 避免读取过多无关内容导致输出过长

### Step 3: 总结并输出可执行建议

#### ⚠️ 内容限制（重要）

- **总结输出内容不超过100行**
- 保持简洁精炼，突出核心要点
- 提供关键修复建议与文档路径

## 问题描述

### 问题表现
1. UI 文本显示为 □□□ 或完全空白
2. Text 组件文字不显示，但 Inspector 中有内容
3. 编译报错找不到 TMPro 命名空间

### 触发条件
- UI 文本内容包含中文/日文/韩文等 CJK 字符，但使用内置字体（不含 CJK 字形）
- 字体使用 `LegacyRuntime.ttf`，但某些 Unity 版本不包含此字体
- 使用了 TextMeshPro 但项目未安装 `com.unity.textmeshpro` 包

## 解决方案

### 关键步骤
1. **所有 UI 文本使用英文**（内置字体不支持 CJK）
2. 字体加载使用 fallback：先尝试 `LegacyRuntime.ttf`，失败则用 `Arial.ttf`
3. 统一使用 `UnityEngine.UI.Text`（Legacy UI），不混用 TextMeshPro（除非确认已安装）

### 关键代码/命令
```csharp
// 字体加载 fallback
Font font = Resources.GetBuiltinResource<Font>("LegacyRuntime.ttf");
if (font == null) font = Resources.GetBuiltinResource<Font>("Arial.ttf");
text.font = font;

// ❌ 不要写中文
text.text = "游戏结束";  // → 显示为 □□□

// ✅ 用英文
text.text = "GAME OVER";
```

### 最终方案
UI 文本统一用英文 + `UnityEngine.UI.Text` + 内置字体 fallback。如需中文支持，必须导入包含 CJK 字形的字体文件（如 Noto Sans CJK）并在运行时加载。不要在 Editor 脚本中使用 `CreateDynamicFontFromOSFont`（无法序列化）。
